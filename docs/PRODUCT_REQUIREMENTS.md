# TodoApp 产品需求文档 v1.0

## 一、产品概述

### 1.1 产品定位
一款轻量、美观、iOS+Windows无缝互通的TodoList应用。核心差异化价值：
- **国内服务器秒级同步**：不用忍受海外服务延迟
- **免费版无核心功能限制**：无限任务/清单/标签/同步
- **原生级Windows体验**：非Web壳，真正的原生应用
- **生活平衡视角（V3.0）**：不仅"做更多"，更关注"做对的事"

### 1.2 目标用户

| 用户画像 | 典型场景 | 核心诉求 |
|---|---|---|
| 职场白领 | 跨设备管理工作任务，Windows办公+iOS通勤 | 多端同步、快速录入、日历视图 |
| 学生 | 作业/考试/社团多项目管理 | 截止日期提醒、子任务拆解 |
| 自由职业者 | 自我管理、习惯养成 | 习惯追踪、番茄钟、数据统计 |
| 生活规划者 | 购物清单/旅行计划/家务安排 | 共享协作、简单快捷 |

### 1.3 产品使命
帮助1亿用户在不焦虑的状态下，高效完成真正重要的事。

## 二、功能需求（User Story 格式）

### 2.1 MVP (V1.0) — 必须交付

#### F-001 用户注册与登录
- **As** 新用户
- **I want** 用邮箱/手机号注册账号
- **So that** 我的数据可以在多设备间同步

**验收标准：**
- [ ] 支持邮箱+验证码注册
- [ ] 支持手机号+短信验证码注册
- [ ] 支持 Magic Link 一键登录
- [ ] 密码长度>=8位，含大小写+数字
- [ ] 注册后自动生成默认清单"收集箱"
- [ ] 登录态保持30天（JWT Refresh Token）

#### F-002 任务创建
- **As** 用户
- **I want** 快速创建待办任务
- **So that** 我不会忘记要做什么事

**验收标准：**
- [ ] 点击FAB打开快速创建面板
- [ ] 输入标题（必填，最大200字）
- [ ] 可选设置：截止日期、优先级(P1-P4)、所属清单、备注
- [ ] 支持自然语言输入："明天下午3点开会 p1" → 自动解析日期和优先级
- [ ] 创建成功后列表即时刷新
- [ ] 离线状态下也能创建，联网后自动同步

#### F-003 任务管理
- **As** 用户
- **I want** 编辑、完成、删除任务
- **So that** 我能管理任务的生命周期

**验收标准：**
- [ ] 点击任务打开详情面板
- [ ] 可编辑标题、备注、日期、优先级、清单
- [ ] 点击圆圈标记完成/取消完成
- [ ] 完成任务显示删除线+淡出动画
- [ ] 已完成任务移到底部（或隐藏）
- [ ] 左滑任务显示删除按钮（需二次确认）
- [ ] 支持批量选择/批量完成/批量删除
- [ ] 撤销删除（5秒SnackBar + 撤销按钮）

#### F-004 子任务
- **As** 用户
- **I want** 把大任务拆解为子任务
- **So that** 复杂任务更容易推进

**验收标准：**
- [ ] 任务详情中可添加子任务
- [ ] 子任务支持独立完成状态
- [ ] 父任务显示子任务完成进度(2/5)
- [ ] 子任务最多嵌套1层（不做无限层级）

#### F-005 多清单管理
- **As** 用户
- **I want** 用不同清单归类任务
- **So that** 工作和生活不会混在一起

**验收标准：**
- [ ] 创建/编辑/删除清单
- [ ] 清单可设置名称和颜色（8色可选）
- [ ] 清单列表支持拖拽排序
- [ ] 默认清单：收集箱、工作、个人
- [ ] 可将清单归档（不删除，隐藏）

#### F-006 优先级与截止日期
- **As** 用户
- **I want** 标记任务的优先级和截止时间
- **So that** 我知道哪些事要先做、什么时候截止

**验收标准：**
- [ ] 4级优先级：P1(紧急)/P2(重要)/P3(普通)/P4(低)
- [ ] 优先级用颜色区分（红/橙/蓝/灰）
- [ ] 设置截止日期（日期+可选时间）
- [ ] 已过期任务日期变红
- [ ] 今天到期任务日期变橙

#### F-007 今日视图
- **As** 用户
- **I want** 打开App就看到今天要做的事
- **So that** 我能快速进入工作状态

**验收标准：**
- [ ] 默认Tab为"今天"
- [ ] 聚合所有清单中今天到期+已过期的任务
- [ ] 空状态显示鼓励性插画+引导
- [ ] 全部完成时触发庆祝动画
- [ ] 支持手动将任意任务添加到"今天"

#### F-008 提醒通知
- **As** 用户
- **I want** 在到期时收到提醒
- **So that** 我不会错过重要任务

**验收标准：**
- [ ] 任务可设置提醒时间（提前5分/15分/30分/1小时/自定义）
- [ ] iOS使用APNs推送
- [ ] Windows使用本地Toast通知
- [ ] 提醒包含任务标题和到期时间
- [ ] 重复任务自动计算下次提醒时间

#### F-009 重复任务
- **As** 用户
- **I want** 设置周期性任务
- **So that** 我不需要每次都手动创建

**验收标准：**
- [ ] 支持：每天/每周/每月/每年/自定义Cron
- [ ] 完成当前实例后自动生成下一个
- [ ] 支持结束条件：永不/指定日期/完成次数

#### F-010 数据同步
- **As** 用户
- **I want** 在iPhone上添加的任务，Windows上也能看到
- **So that** 我的数据随时一致

**验收标准：**
- [ ] 注册登录后自动开启同步
- [ ] 离线操作暂存本地，联网后自动合并
- [ ] 同步冲突自动处理（CRDT合并）
- [ ] 同步状态指示器（已同步/同步中/未连接）
- [ ] 同步延迟 < 2秒（国内网络）

#### F-011 搜索
- **As** 用户
- **I want** 通过关键词快速找到任务
- **So that** 不用在清单里翻找

**验收标准：**
- [ ] 全局搜索（标题+备注）
- [ ] 实时建议（输入即搜索）
- [ ] 搜索结果按清单分组
- [ ] 点击结果跳转到对应任务

## 三、非功能需求

### 3.1 性能要求

| 指标 | 目标 |
|---|---|
| App冷启动时间 | < 1.5s (iOS) / < 2s (Windows) |
| 任务列表滚动 | 60fps，支持1000+任务无卡顿 |
| 同步延迟 | < 2s（正常网络） |
| 离线操作响应 | < 50ms 本地写入 |
| API响应时间 | p50 < 100ms，p99 < 500ms |
| 内存占用 | < 150MB (iOS) / < 300MB (Windows) |
| 安装包大小 | < 80MB (iOS) / < 120MB (Windows) |

### 3.2 安全要求
- 数据传输全程 TLS 1.3 加密
- 密码使用 bcrypt(12 rounds) 哈希存储
- JWT Token 过期策略：Access Token 30min / Refresh Token 30d
- 用户数据逻辑隔离，API做权限校验
- SQL注入防护（参数化查询）
- 敏感操作日志审计
- 隐私政策合规（GDPR / 个人信息保护法）

### 3.3 兼容性要求

| 平台 | 最低版本 |
|---|---|
| iOS | iOS 16.0+ |
| Windows | Windows 10 1903+ / Windows 11 |

### 3.4 可用性要求
- 目标用户首次使用3分钟内完成第一个任务
- 核心操作（创建/完成/删除）不超过3步
- 错误提示使用用户能理解的语言（非技术术语）
- 所有操作支持撤销（5秒内）

## 四、用户流程

### 4.1 核心流程：创建并完成任务

```
[打开App]
    │
    ▼
[今日视图] ──空?──▶ [空状态引导] ──▶ [新建任务]
    │                                    │
    │ 有任务                              │
    ▼                                    ▼
[任务列表]  ◀────────────── [快速创建面板]
    │                          │
    │ 点击圆圈                   │ 输入标题+日期+优先级
    ▼                          │
[完成动画]                     │
    │                          │
    ▼                          │
[任务移到已完成区域] ◀─────────┘
    │
    ▼ (3秒内可点"撤销")
[任务恢复]
```

### 4.2 核心流程：多端同步

```
[设备A: iOS]                    [服务器]                    [设备B: Windows]
    │                              │                              │
    │ 新建任务"买咖啡豆"             │                              │
    │─────────────▶                │                              │
    │  {op: create, data: task}    │                              │
    │                              │  CRDT合并→持久化              │
    │              ◀───────────────│                              │
    │              ack: success    │                              │
    │                              │  WebSocket Push              │
    │                              │─────────────────────────▶    │
    │                              │    {task_id, data}            │
    │                              │                              │ 更新本地DB
    │                              │                              │ 刷新UI
```

### 4.3 核心流程：每日回顾

```
[打开App] → [今日视图]
    │
    ├─ 全部完成? → [🎉 庆祝动画] → [引导查看统计/设置明日计划]
    │
    ├─ 有未完成? → [按优先级排列]
    │     │
    │     ├─ 有已过期? → [红色高亮提醒] → [用户选择: 今天做完/推迟/删除]
    │     │
    │     └─ 正常处理 → [逐项完成]
    │
    └─ 无任务? → [空状态+时间线建议: "早上适合做深度工作/下午处理沟通"]
```

### A. 术语表

| 术语 | 定义 |
|---|---|
| 清单 (List) | 任务的容器/分类，如"工作""个人" |
| 任务 (Task) | 一个待办事项，包含标题、日期、优先级等 |
| 子任务 (Subtask) | 任务下的子项，支持独立完成状态 |
| 今日视图 (Today) | 聚合所有今日到期+已过期任务的视图 |
| CRDT | Conflict-free Replicated Data Type，无冲突数据同步算法 |
| FAB | Floating Action Button，浮动操作按钮 |
