# MingDay Project Memory

> 本文件是 MingDay 项目的最高优先级产品上下文入口。若本文与历史规划文档存在冲突，默认以本文为准；`PRODUCT_REQUIREMENTS.md`、`DESIGN_SPEC.md`、`DEVELOPMENT_DOC.md` 作为详细资料和后续路线图参考。

## 当前决策

- 正式产品名：MingDay。
- 产品类型：轻量、美观、低焦虑的 TodoList App。
- 第一版策略：精简 MVP，先完成单人、离线优先、任务闭环体验。
- 首版不追求完整生产级账号、跨端同步、AI、协作或复杂增长功能。
- 历史文档中的 `TodoApp` 视为旧占位名，后续新增文档和代码命名优先使用 `MingDay`。
- 当前仓库根 README 仍是 DouTracker，后续应单独安排项目命名清理任务，不在 MVP 记忆落地时直接删除旧代码。

## 品牌与体验

- Logo 气质：清晨、轻量、柔和、完成感。
- 核心关键词：明亮、安静、可信赖、不过度制造焦虑。
- 视觉方向延续 Logo 的白底、蓝色完成感、暖黄色清晨感。
- 红色只用于过期、删除等危险或异常状态，不用于制造未完成数量焦虑。
- 完成任务应带来轻量满足感，例如勾选、淡出、柔和庆祝反馈。

## 目标用户

- 职场白领：在 Windows 办公和 iOS 通勤之间管理工作任务。
- 学生：管理作业、考试、社团和项目截止日期。
- 自由职业者：拆解工作、维持节奏、做轻量自我管理。
- 生活规划者：管理购物、旅行、家务等低负担清单。

## MVP 范围

### 核心任务

- 创建任务：标题必填，支持备注、截止日期、优先级、所属清单。
- 管理任务：编辑、完成/取消完成、删除、5 秒内撤销删除。
- 子任务：任务详情内添加一层子任务，显示完成进度。

### 信息组织

- 默认清单：收集箱、工作、个人。
- 清单支持创建、编辑名称、选择颜色、归档。
- 优先级 P1/P2/P3/P4 使用红、橙、蓝、灰语义色。

### 首屏体验

- 默认进入“今天”视图。
- 今天视图聚合今天到期、已过期、手动加入今天的任务。
- 空状态引导创建任务。
- 全部完成时显示轻量庆祝反馈。

### 本地能力

- 首版使用本地持久化作为唯一数据源。
- 所有任务操作必须先保证离线可用。
- 搜索支持标题和备注关键词匹配。
- 数据模型预留 `sync_status`、`updated_at`、`deleted_at` 等字段，方便后续同步。

## 暂不做事项

- 账号注册与登录。
- 多端实时同步和服务端 CRDT 合并。
- AI 智能建议、智能拆解、任务量健康评估。
- 协作共享、成员动态、评论讨论。
- 番茄钟、习惯追踪、完整统计报表。
- iOS Widget、Siri 快捷指令、上架流程。
- 生产级推送；提醒通知可在 MVP 后半阶段先做本地提醒。

## 后续路线

- V1 后半阶段：本地提醒、重复任务基础版、可用性打磨、数据导入导出。
- V2：标签、日历视图、番茄钟、习惯追踪、数据统计、深色模式。
- V3：生活平衡轮、AI 智能建议、协作共享、iOS Widget、Siri 快捷指令。
- 云同步应在本地数据模型和任务闭环稳定后单独设计，不应阻塞精简 MVP。

## 实施顺序

1. 项目命名清理、App 壳、主题 token、底部导航、今天/清单/设置基础页面。
2. 任务与清单本地模型、CRUD、任务详情面板、完成与删除撤销。
3. 今天视图聚合、优先级与日期状态、搜索、空状态和完成反馈。
4. 本地提醒、重复任务基础版、可用性打磨与测试。

## 技术原则

- 离线优先：所有用户操作先写入本地，再考虑后续同步。
- 数据演进友好：任务、清单、子任务实体保留更新时间、软删除和同步状态字段。
- 核心操作不超过 3 步：创建、完成、删除必须直接、清晰、可撤销。
- 保持实现收敛：首版不为了后续云同步过早引入复杂服务端架构。
- UI 需兼顾小屏和桌面宽屏，避免文字溢出、控件重叠和过度装饰。

## 首版成功标准

- 新用户首次打开 3 分钟内可以创建第一个任务。
- 创建、完成、删除均不超过 3 步。
- 离线创建 10 个任务，重启应用后数据仍存在。
- 1000 条任务列表滚动保持流畅。
- 删除任务后 5 秒内可撤销，超过后不可恢复。
- 用户能从今天视图清楚知道现在该做什么。

## 测试要求

- 单元测试覆盖任务创建、编辑、完成、删除、撤销。
- 单元测试覆盖今天视图筛选规则：今天到期、已过期、手动加入今天。
- 单元测试覆盖优先级排序、清单归档、搜索匹配。
- UI/交互测试覆盖空状态、新建任务、任务详情、完成动画、删除撤销。
- 布局测试覆盖小屏和桌面宽屏，确保不重叠、不溢出。
